`Volo.Abp.BlobStoring.Google` is the GCS backend for ABP's BLOB Storing system. It wraps the official `Google.Cloud.Storage.V1` SDK, supports both service‑account JSON keys and Application Default Credentials (ADC), and follows the same tenant‑prefix layout as the Azure and AWS providers.

Source: `framework/src/Volo.Abp.BlobStoring.Google/Volo/Abp/BlobStoring/Google/`. The provider itself is `GoogleBlobProvider`; configuration is in `GoogleBlobProviderConfiguration`; the per‑request `StorageClient` is constructed on the fly inside the provider.

## Module and configuration

Add `AbpBlobStoringGoogleModule` to your application module's `[DependsOn]` and configure any container with `UseGoogle(...)`.

```csharp
[DependsOn(
    typeof(AbpBlobStoringModule),
    typeof(AbpBlobStoringGoogleModule)
)]
public class MyAppModule : AbpModule
{
    public override void ConfigureServices(ServiceConfigurationContext context)
    {
        Configure<AbpBlobStoringOptions>(options =>
        {
            options.Containers.ConfigureDefault(container =>
            {
                container.UseGoogle(google =>
                {
                    google.ProjectId = "my-gcp-project";
                    google.ClientEmail = "blob-sa@my-gcp-project.iam.gserviceaccount.com";
                    google.PrivateKey = "-----BEGIN PRIVATE KEY-----\n…\n-----END PRIVATE KEY-----";
                    google.ContainerName = "myapp-bucket";
                    google.CreateContainerIfNotExists = true;
                });
            });

            // Or, on Cloud Run / GKE / Cloud Functions where ADC is available:
            options.Containers.Configure<InvoiceContainer>(container =>
            {
                container.UseGoogle(google =>
                {
                    google.ProjectId = "my-gcp-project";
                    google.UseApplicationDefaultCredentials = true;
                    google.ContainerName = "myapp-invoices";
                });
            });
        });
    }
}
```

`UseGoogle` mirrors the other providers: it sets the provider type, adds the Google naming normalizer, and exposes a typed view.

```csharp title="framework/src/Volo.Abp.BlobStoring.Google/Volo/Abp/BlobStoring/Google/GoogleBlobContainerConfigurationExtensions.cs"
public static BlobContainerConfiguration UseGoogle(
    this BlobContainerConfiguration containerConfiguration,
    Action<GoogleBlobProviderConfiguration> googleConfigureAction)
{
    containerConfiguration.ProviderType = typeof(GoogleBlobProvider);
    containerConfiguration.NamingNormalizers.TryAdd<GoogleBlobNamingNormalizer>();

    googleConfigureAction(new GoogleBlobProviderConfiguration(containerConfiguration));

    return containerConfiguration;
}
```

## `GoogleBlobProviderConfiguration`

```csharp title="framework/src/Volo.Abp.BlobStoring.Google/Volo/Abp/BlobStoring/Google/GoogleBlobProviderConfiguration.cs"
public class GoogleBlobProviderConfiguration
{
    /// <summary>
    /// Unique identifier for your project. See
    /// https://cloud.google.com/resource-manager/docs/creating-managing-projects
    /// </summary>
    public string? ProjectId { get; set; }

    /// <summary>Service-account email generated by GCP.</summary>
    public string? ClientEmail { get; set; }

    /// <summary>
    /// Private key generated by GCP. Starts with '-----BEGIN PRIVATE KEY-----'
    /// and ends with '-----END PRIVATE KEY-----'.
    /// </summary>
    public string? PrivateKey { get; set; }

    /// <summary>OAuth 2.0 scopes.</summary>
    public List<string>? Scopes { get; set; }

    /// <summary>
    /// Use the application default credentials.
    /// https://cloud.google.com/docs/authentication/provide-credentials-adc
    /// Default: false.
    /// </summary>
    public bool UseApplicationDefaultCredentials { get; set; }

    /// <summary>
    /// GCS bucket name. If not set, the BlobProviderArgs.ContainerName is used.
    /// </summary>
    public string? ContainerName { get; set; }

    /// <summary>Default: false.</summary>
    public bool CreateContainerIfNotExists { get; set; }
}
```

Two credential modes are supported:

| Mode | Flag | What you set |
| --- | --- | --- |
| Service account JSON | (default) | `ClientEmail`, `PrivateKey`, optional `Scopes` |
| Application Default Credentials | `UseApplicationDefaultCredentials = true` | `ProjectId` only; everything else is resolved from the environment |

ADC is the right choice on Cloud Run, GKE Workload Identity, Cloud Functions, and any GCP runtime that injects credentials automatically.

## `GoogleBlobProvider`

`GoogleBlobProvider` constructs a `StorageClient` per call (cheap; it does not eagerly authenticate) and dispatches to `UploadObjectAsync`, `DownloadObjectAsync`, `DeleteObjectAsync`, `GetObjectAsync`.

```csharp title="framework/src/Volo.Abp.BlobStoring.Google/Volo/Abp/BlobStoring/Google/GoogleBlobProvider.cs"
public async override Task SaveAsync(BlobProviderSaveArgs args)
{
    var configuration = args.Configuration.GetGoogleConfiguration();
    var storageClient = await GetStorageClientClientAsync(args);
    var blobName = GoogleBlobNameCalculator.Calculate(args);
    var containerName = GetContainerName(args);

    if (await BlobExistsAsync(args, blobName) && !args.OverrideExisting)
    {
        throw new BlobAlreadyExistsException(
            $"Saving BLOB '{args.BlobName}' does already exists in the container '{GetContainerName(args)}'! " +
            $"Set {nameof(args.OverrideExisting)} if it should be overwritten.");
    }

    if (configuration.CreateContainerIfNotExists)
    {
        await CreateContainerIfNotExists(args);
    }

    await storageClient.UploadObjectAsync(
        containerName, blobName, contentType: "application/octet-stream", args.BlobStream);
}
```

`GetOrNullAsync` returns a fresh `MemoryStream` rewound to `0`:

```csharp
public async override Task<Stream?> GetOrNullAsync(BlobProviderGetArgs args)
{
    var storageClient = await GetStorageClientClientAsync(args);
    var blobName = GoogleBlobNameCalculator.Calculate(args);
    var containerName = GetContainerName(args);

    if(!await BlobExistsAsync(args, blobName)) return null;

    var stream = new MemoryStream();
    await storageClient.DownloadObjectAsync(containerName, blobName, stream);
    stream.Seek(0, SeekOrigin.Begin);
    return stream;
}
```

Because the SDK downloads to a writable target, the implementation buffers the whole object in memory. For large blobs, override the provider and use `DownloadObjectAsync` against a different `Stream` (for example, the HTTP response stream of an MVC `FileResult`) to avoid double‑buffering.

### `StorageClient` creation

```csharp
protected virtual async Task<StorageClient> GetStorageClientClientAsync(BlobProviderArgs args)
{
    var configuration = args.Configuration.GetGoogleConfiguration();
    if (configuration.UseApplicationDefaultCredentials)
    {
        return await StorageClient.CreateAsync();
    }

    var googleCredential = GoogleCredential.FromServiceAccountCredential(
        new ServiceAccountCredential(
            new ServiceAccountCredential.Initializer(configuration.ClientEmail)
                {
                    ProjectId = configuration.ProjectId,
                    Scopes = configuration.Scopes
                }
                .FromPrivateKey(configuration.PrivateKey)
        ));

    return await StorageClient.CreateAsync(googleCredential);
}
```

The service‑account path builds a `ServiceAccountCredential` from the email + private key (the same fields you find in a `key.json`) and wraps it in `GoogleCredential`. For most use cases the default scopes are sufficient; override only when you need write‑restricted scopes for a least‑privilege service account.

### Bucket auto‑creation

```csharp
protected virtual async Task CreateContainerIfNotExists(BlobProviderArgs args)
{
    var storageClient = await GetStorageClientClientAsync(args);
    var configuration = args.Configuration.GetGoogleConfiguration();
    if (await ContainerExistsAsync(args, storageClient)) return;

    await storageClient.CreateBucketAsync(configuration.ProjectId, GetContainerName(args));
}
```

The `ProjectId` configuration value is *required* for `CreateBucketAsync` even when ADC is used, because buckets always belong to an explicit project.

### Existence check pattern

```csharp
protected virtual async Task<bool> BlobExistsAsync(BlobProviderArgs args, string blobName)
{
    var storageClient = await GetStorageClientClientAsync(args);
    if(!await ContainerExistsAsync(args, storageClient)) return false;

    try
    {
        await storageClient.GetObjectAsync(GetContainerName(args), blobName);
    }
    catch (GoogleApiException e) when (e.HttpStatusCode == HttpStatusCode.NotFound)
    {
        return false;
    }
    return true;
}
```

Two REST calls every time, just like the AWS provider: a bucket check followed by an object check. For high‑throughput access patterns where the bucket is known to exist, override `BlobExistsAsync` in a subclass.

## Tenant naming — `DefaultGoogleBlobNameCalculator`

The Google provider uses the same tenant prefix logic as Azure and AWS. Object keys look like `host/<blobName>` for the host or `tenants/<tenantId>/<blobName>` for a tenant. The blob name is set by `GoogleBlobNameCalculator` and concatenated into the object key sent to GCS.

Combined with `BlobContainerConfiguration.IsMultiTenant`, this gives per‑tenant logical folders inside one bucket. See [/tenancy/multi-tenancy-core](/tenancy/multi-tenancy-core) for the `ICurrentTenant` mechanics that drive the calculator.

## `GoogleBlobNamingNormalizer`

GCS bucket names: 3–63 chars (up to 222 with dots), lowercase letters/digits/hyphens/underscores/dots, no `goog` prefix, no `google` substring. `GoogleBlobNamingNormalizer` rewrites ABP container names to comply with these rules before the provider receives them. Object names (blob keys) are much more permissive and are passed through.

## Operational guidance

<AccordionGroup>
  <Accordion title="Choose ADC when running on GCP" icon="cloud">
    On Cloud Run, GKE (with Workload Identity), and Cloud Functions, set `UseApplicationDefaultCredentials = true` and leave `ClientEmail`/`PrivateKey` empty. You get rotation, audit, and per‑pod identity without code changes.
  </Accordion>
  <Accordion title="Keep PrivateKey out of source control" icon="key">
    The service‑account `PrivateKey` is sensitive. Bind it from environment variables, Secret Manager, or another configuration provider — never check it into `appsettings.json`.
  </Accordion>
  <Accordion title="ProjectId is required for bucket creation" icon="folder-plus">
    Even with `UseApplicationDefaultCredentials = true`, `CreateContainerIfNotExists = true` requires `ProjectId` to be set. ADC can supply credentials but not the bucket's owning project.
  </Accordion>
  <Accordion title="Object versioning" icon="clock-rotate-left">
    Bucket‑level versioning is a GCS feature, not a provider feature. Enable it on the bucket via IaC; the provider's `OverrideExisting` simply maps to the SDK's overwrite semantics.
  </Accordion>
  <Accordion title="Large downloads buffer in memory" icon="memory">
    `GetOrNullAsync` downloads into a `MemoryStream`. For very large objects, override the provider and stream directly to the response.
  </Accordion>
</AccordionGroup>

## Usage example

```csharp
[BlobContainerName("user-uploads")]
public class UserUploadContainer { }

public class UserUploadAppService : ApplicationService
{
    private readonly IBlobContainer<UserUploadContainer> _blobs;

    public UserUploadAppService(IBlobContainer<UserUploadContainer> blobs)
    {
        _blobs = blobs;
    }

    public Task SaveAsync(string fileName, Stream content)
        => _blobs.SaveAsync(fileName, content, overrideExisting: true);

    public Task<Stream?> GetOrNullAsync(string fileName)
        => _blobs.GetOrNullAsync(fileName);
}
```

With the configuration shown at the top, an upload by tenant `xyz` lands at:

```text
GCS bucket:  myapp-bucket
Object name: tenants/<tenantId>/<fileName>
```

## Comparison with other cloud providers

| Aspect | [Google](/blob/google) | [Azure](/blob/azure) | [AWS](/blob/aws) |
| --- | --- | --- | --- |
| SDK | `Google.Cloud.Storage.V1` | `Azure.Storage.Blobs` | `AWSSDK.S3` |
| Credential modes | Service account JSON, ADC | Connection string | Static, profile, STS session, STS federation |
| Bucket auto-create | `CreateContainerIfNotExists` | `CreateContainerIfNotExists` | `CreateContainerIfNotExists` |
| Existence check | bucket + object | container + blob | bucket + object |
| Tenant prefix | `host/`, `tenants/{id}/` | `host/`, `tenants/{id}/` | `host/`, `tenants/{id}/` |
| Download buffering | `MemoryStream` | streaming | streaming |

All three providers share the same `BlobProviderBase` contract. The differences are which SDK they wrap and which features the underlying service offers.

## Migration notes

To move data from another provider into GCS, the simplest path is:

1. Provision the GCS bucket out‑of‑band.
2. Use `gsutil rsync` or `gcloud storage cp` to copy existing object keys from the old backend, preserving the `host/` / `tenants/{id}/` prefix layout.
3. Switch the container's `Use*(...)` call to `UseGoogle(...)` in your application module and redeploy.

Because the tenant prefix layout is identical across Azure, AWS, GCS, Aliyun, MinIO and Bunny, no code changes are needed and existing blob keys keep working.
