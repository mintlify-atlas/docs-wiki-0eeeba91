`Volo.Abp.AspNetCore.Mvc.UI.Bootstrap` ships Bootstrap 5 as a typed, server-rendered HTML library. Instead of writing the markup by hand —

```html
<button type="button" class="btn btn-primary">Save</button>
```

— you write

```html
<abp-button button-type="Primary" text="Save" icon="save" />
```

and the tag helper outputs the correct Bootstrap classes, ARIA attributes, icon span, busy-state markup, and so on. The package contains **no JavaScript and no CSS of its own** — it is a thin, opinionated tag-helper layer on top of Bootstrap that every ABP theme (Basic, LeptonX, custom) uses to render its views. The actual Bootstrap CSS/JS is delivered through the bundling/contributor pipeline (see [/aspnetcore/mvc-ui-packages](/aspnetcore/mvc-ui-packages)).

## Module

The Bootstrap module is small — it depends on `AbpAspNetCoreMvcUiModule` and only registers its embedded views (each tag helper's `*.cshtml` partial lives in the assembly):

```csharp
[DependsOn(typeof(AbpAspNetCoreMvcUiModule))]
public class AbpAspNetCoreMvcUiBootstrapModule : AbpModule
{
    public override void ConfigureServices(ServiceConfigurationContext context)
    {
        Configure<AbpVirtualFileSystemOptions>(options =>
        {
            options.FileSets.AddEmbedded<AbpAspNetCoreMvcUiBootstrapModule>(
                "Volo.Abp.AspNetCore.Mvc.UI.Bootstrap");
        });
    }
}
```

It is automatically included by `AbpAspNetCoreMvcUiThemeSharedModule` (see [/ui/theme-shared](/ui/theme-shared)), so every theme has Bootstrap available without any extra wiring.

## The tag helper base class

Every Bootstrap tag helper is a thin POCO that exposes properties; the actual HTML generation lives in a companion `*TagHelperService`. Both inherit from common bases:

```csharp
// TagHelpers/AbpTagHelper.cs (shape)
public abstract class AbpTagHelper<TTagHelper, TService> : TagHelper
    where TTagHelper : AbpTagHelper<TTagHelper, TService>
    where TService : AbpTagHelperService<TTagHelper>
{
    protected TService TagHelperService { get; }

    protected AbpTagHelper(TService tagHelperService)
    {
        TagHelperService = tagHelperService;
        TagHelperService.SetTagHelper((TTagHelper)this);
    }

    public override Task ProcessAsync(TagHelperContext context, TagHelperOutput output)
        => TagHelperService.ProcessAsync(context, output);
}
```

This split exists so that **localization** (`IAbpTagHelperLocalizer`) and DI services (icon types, model binders) can be injected on the service side without polluting the tag-helper signature.

## Examples

### Button

```csharp
[HtmlTargetElement("abp-button", TagStructure = TagStructure.NormalOrSelfClosing)]
public class AbpButtonTagHelper
    : AbpTagHelper<AbpButtonTagHelper, AbpButtonTagHelperService>, IButtonTagHelperBase
{
    public AbpButtonType ButtonType { get; set; } = AbpButtonType.Default;
    public AbpButtonSize Size { get; set; } = AbpButtonSize.Default;
    public string? BusyText { get; set; }
    public string? Text { get; set; }
    public string? Icon { get; set; }
    public bool? Disabled { get; set; }
    public FontIconType IconType { get; set; } = FontIconType.FontAwesome;
    public bool BusyTextIsHtml { get; set; }
}
```

```html
<abp-button button-type="Primary"
            icon="save"
            text="@L["Save"]"
            busy-text="@L["SavingWithThreeDot"]" />
```

`AbpButtonType` lives in the same folder — `Default`, `Primary`, `Secondary`, `Success`, `Danger`, `Warning`, `Info`, `Light`, `Dark`, `Link`, `Outline_*`. `FontIconType` lets you flip between `FontAwesome` (default) and `BootstrapIcon`.

### Input

```csharp
public class AbpInputTagHelper : AbpTagHelper<AbpInputTagHelper, AbpInputTagHelperService>
{
    public ModelExpression AspFor { get; set; } = default!;
    public string? Label { get; set; }
    public string? LabelTooltip { get; set; }
    [HtmlAttributeName("info")]      public string? InfoText { get; set; }
    [HtmlAttributeName("disabled")]  public bool IsDisabled { get; set; }
    [HtmlAttributeName("readonly")]  public bool? IsReadonly { get; set; }
    public bool AutoFocus { get; set; }
    [HtmlAttributeName("type")]      public string? InputTypeName { get; set; }
    public AbpFormControlSize Size { get; set; } = AbpFormControlSize.Default;
    [HtmlAttributeName("required-symbol")] public bool DisplayRequiredSymbol { get; set; } = true;
    [HtmlAttributeName("asp-format")]      public string? Format { get; set; }
    public string? Name { get; set; }
    public string? Value { get; set; }
    public bool SuppressLabel { get; set; }
    [HtmlAttributeName("floating-label")]  public bool FloatingLabel { get; set; }
    public CheckBoxHiddenInputRenderMode? CheckBoxHiddenInputRenderMode { get; set; }
    [HtmlAttributeName("use-switch-checkbox")] public bool UseSwitchCheckBox { get; set; }
    public bool AddMarginBottomClass { get; set; } = true;
}
```

```html
<abp-input asp-for="Title" auto-focus="true" />
<abp-input asp-for="PublishDate" type="date" />
<abp-input asp-for="IsPublished" use-switch-checkbox="true" />
<abp-input asp-for="Email" floating-label="true" info="Used for password recovery." />
```

`AbpInputTagHelperService` inspects the bound `ModelExpression` to drive the rendered `<input type="...">`, the validation attributes, the floating-label markup, and the label text (pulled from `DisplayAttribute` / `DisplayName` / the resource).

### Table, Tabs, Nav, Dropdown

The other folders follow the same pattern:

- `Table/` — `<abp-table>` plus header-scope helpers (`AbpTableHeadScopeTagHelper`) that handle `scope="col"` / `scope="row"` accessibility.
- `Tab/` — `<abp-tabs>` wraps `<abp-tab>` and `<abp-tab-link>` children. Supports `<abp-tab-dropdown>` for nested menus.
- `Nav/` — building blocks for the top navbar (`AbpNavbarBrandTagHelper`, `AbpNavLinkTagHelper`).
- `Dropdown/` — split buttons, header items, dividers, sub-items.

### Table

```html
<abp-table striped-rows="true" hoverable-rows="true">
    <thead>
        <tr>
            <abp-table-header-scope>Title</abp-table-header-scope>
            <abp-table-header-scope>Author</abp-table-header-scope>
            <abp-table-header-scope>Price</abp-table-header-scope>
        </tr>
    </thead>
    <tbody>
        @foreach (var book in Model.Books)
        {
            <tr>
                <td>@book.Title</td>
                <td>@book.Author</td>
                <td>@book.Price.ToString("C")</td>
            </tr>
        }
    </tbody>
</abp-table>
```

`AbpTableHeadScopeTagHelper` renders `<th scope="col">` / `<th scope="row">` automatically — a small accessibility win you get for free across every ABP module's admin UI.

## How themes use it

Every theme's layout is a composition of these tag helpers. Look at [`modules/basic-theme`](/ui/basic-theme) `Themes/Basic/Layouts/Application.cshtml`, the main navbar component, or `Themes/Basic/Components/Toolbar/UserMenu/*.cshtml` — they are entirely written in `<abp-nav-*>`, `<abp-dropdown-*>`, and `<abp-button>` markup.

The same is true of every ABP module's admin UI (Identity, Tenant Management, OpenIddict, Feature Management, CMS Kit, …): the entire CRUD UI is `<abp-card>` + `<abp-table>` + `<abp-modal>` + `<abp-input>`.
