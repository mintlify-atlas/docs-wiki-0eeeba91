ABP's framework packages expose dozens of `Abp*Options` types, but a small subset accounts for almost every line of `Configure<>` you will ever write. This page is the catalogue: one row per options class, with the source file, the namespace, and the headline knobs.

For the *mechanics* of how `Configure<>`, `PreConfigure<>`, and `PostConfigure<>` work — and what `AbpOptionsFactory<T>` does behind the scenes — see [`/core/options-and-configuration`](/core/options-and-configuration). For where `IConfiguration` comes from inside a module see [`/config/overview`](/config/overview).

## AbpAuditingOptions

Auditing's master switch and its contributor lists.

```csharp
// framework/src/Volo.Abp.Auditing/Volo/Abp/Auditing/AbpAuditingOptions.cs
public class AbpAuditingOptions
{
    /// <summary>
    /// If this value is true, auditing will not throw an exceptions and it will
    /// log it when an error occurred while saving AuditLog. Default: true.
    /// </summary>
    public bool HideErrors { get; set; }

    /// <summary> Default: true. </summary>
    public bool IsEnabled { get; set; }

    /// <summary> The name of the application or service writing audit logs. </summary>
    public string? ApplicationName { get; set; }

    /// <summary> Default: true. </summary>
    public bool IsEnabledForAnonymousUsers { get; set; }

    /// <summary> Audit log on exceptions. Default: true. </summary>
    public bool AlwaysLogOnException { get; set; }
    // … Contributors, EntityHistorySelectors, IgnoredTypes, etc.
}
```

Typical wiring:

```csharp
Configure<AbpAuditingOptions>(options =>
{
    options.ApplicationName  = "MyProjectName.HttpApi.Host";
    options.IsEnabled        = true;
    options.IsEnabledForAnonymousUsers = false;
    options.EntityHistorySelectors.AddAllEntities();
});
```

## AbpBlobStoringOptions

Tiny class that delegates everything to `BlobContainerConfigurations` — a dictionary of named containers, each with its own provider.

```csharp
// framework/src/Volo.Abp.BlobStoring/Volo/Abp/BlobStoring/AbpBlobStoringOptions.cs
public class AbpBlobStoringOptions
{
    public BlobContainerConfigurations Containers { get; }

    public AbpBlobStoringOptions()
    {
        Containers = new BlobContainerConfigurations();
    }
}
```

Typical wiring (using the file-system provider as an example):

```csharp
Configure<AbpBlobStoringOptions>(options =>
{
    options.Containers.ConfigureDefault(container =>
    {
        container.UseFileSystem(fileSystem =>
        {
            fileSystem.BasePath = "/var/blobs";
        });
    });
});
```

## AbpDistributedEventBusOptions

Carries the registered handler type list, plus the outbox and inbox configuration dictionaries used by the persisted event bus.

```csharp
// framework/src/Volo.Abp.EventBus/Volo/Abp/EventBus/Distributed/AbpDistributedEventBusOptions.cs
public class AbpDistributedEventBusOptions
{
    public ITypeList<IEventHandler> Handlers { get; }
    public OutboxConfigDictionary Outboxes { get; }
    public InboxConfigDictionary Inboxes { get; }

    public AbpDistributedEventBusOptions()
    {
        Handlers = new TypeList<IEventHandler>();
        Outboxes = new OutboxConfigDictionary();
        Inboxes  = new InboxConfigDictionary();
    }
}
```

```csharp
public class AbpDistributedEntityEventOptions
```

## AbpClockOptions

One knob: the `DateTimeKind` that `IClock.Now` returns. `Utc` is the recommended value for modern services; the default is `Unspecified` for backwards compatibility.

```csharp
// framework/src/Volo.Abp.Timing/Volo/Abp/Timing/AbpClockOptions.cs
public class AbpClockOptions
{
    /// <summary> Default: <see cref="DateTimeKind.Unspecified"/> </summary>
    public DateTimeKind Kind { get; set; }

    public AbpClockOptions()
    {
        Kind = DateTimeKind.Unspecified;
    }
}
```

```csharp
Configure<AbpClockOptions>(options =>
{
    options.Kind = DateTimeKind.Utc;
});
```

See [`/core/timing`](/core/timing) for the `IClock` consumer side.

## AbpDistributedCacheOptions

The cache prefix (so multiple deployments can share one Redis), the error-swallowing flag, the default TTL, and a list of per-cache configurators.

```csharp
// framework/src/Volo.Abp.Caching/Volo/Abp/Caching/AbpDistributedCacheOptions.cs
public class AbpDistributedCacheOptions
{
    public bool HideErrors { get; set; } = true;
    public string KeyPrefix { get; set; }
    public DistributedCacheEntryOptions GlobalCacheEntryOptions { get; set; }
    public List<Func<string, DistributedCacheEntryOptions?>> CacheConfigurators { get; set; }
    // …
}
```

```csharp
Configure<AbpDistributedCacheOptions>(options =>
{
    options.KeyPrefix = "MyApp:";
    options.GlobalCacheEntryOptions = new DistributedCacheEntryOptions
    {
        AbsoluteExpirationRelativeToNow = TimeSpan.FromMinutes(20)
    };
});
```

## AbpEndpointRouterOptions

A list of `Action<EndpointRouteBuilderContext>` callbacks that ABP runs against the endpoint route builder. Used by modules that contribute endpoints (the `account` module adds OpenIddict endpoints this way, for example).

```csharp
// framework/src/Volo.Abp.AspNetCore/Microsoft/AspNetCore/Routing/AbpEndpointRouterOptions.cs
public class AbpEndpointRouterOptions
{
    public List<Action<EndpointRouteBuilderContext>> EndpointConfigureActions { get; }
}
```

```csharp
Configure<AbpEndpointRouterOptions>(options =>
{
    options.EndpointConfigureActions.Add(ctx =>
    {
        ctx.Endpoints.MapHealthChecks("/health");
    });
});
```

## AbpAspNetCoreContentOptions

Owns the MIME-type map used by the [virtual file system](/core/volo-abp-core) when serving embedded static files, plus an opt-in allowlist for extra physical folders/extensions:

```csharp

public class AbpAspNetCoreContentOptions
{
    public Dictionary<string, string> ContentTypeMaps { get; }
    public List<string> AllowedExtraWebContentFolders { get; }
    public List<string> AllowedExtraWebContentFileExtensions { get; }
    // ctor seeds ContentTypeMaps with hundreds of extension→MIME entries
}
```

## AbpAspNetCoreMvcOptions

The single biggest options class in the MVC integration. Controls conventional controllers, integration-service exposure, model validation, and the API-explorer group-name override.

```csharp
// framework/src/Volo.Abp.AspNetCore.Mvc/Volo/Abp/AspNetCore/Mvc/AbpAspNetCoreMvcOptions.cs
public class AbpAspNetCoreMvcOptions
{
    public bool? MinifyGeneratedScript { get; set; }

    public AbpConventionalControllerOptions ConventionalControllers { get; }

    public HashSet<Type> IgnoredControllersOnModelExclusion { get; }
    public HashSet<Type> ControllersToRemove { get; }

    public bool ExposeIntegrationServices { get; set; } = false;
    public bool ExposeClientProxyServices { get; set; } = false;

    public bool AutoModelValidation { get; set; }
    public bool ChangeControllerModelApiExplorerGroupName { get; set; }
    // ctor sets AutoModelValidation = true, ChangeControllerModelApiExplorerGroupName = true
}
```

The very common `PreConfigure<>` pattern that templates ship with:

```csharp
PreConfigure<AbpAspNetCoreMvcOptions>(options =>
{
    options.ConventionalControllers.Create(typeof(MyProjectNameApplicationModule).Assembly);
});
```

## AbpHttpClientBuilderOptions

Hooks that run *while the typed HttpClient is being built* for each dynamic-proxy client. Modules typically `.Add(...)` to these lists to inject a handler or configure a base address.

```csharp
// framework/src/Volo.Abp.Http.Client/Volo/Abp/Http/Client/AbpHttpClientBuilderOptions.cs
public class AbpHttpClientBuilderOptions
{
    public List<Action<string, IHttpClientBuilder>> ProxyClientBuildActions { get; }
    internal HashSet<string> ConfiguredProxyClients { get; }
    public List<Action<string, IServiceProvider, HttpClient>> ProxyClientActions { get; }
    public List<Action<string, IServiceProvider, HttpClientHandler>> ProxyClientHandlerActions { get; }
}
```

## AbpHttpClientOptions

The registered dynamic-proxy clients (one entry per remote-service consumer) plus the pre-send action map.

```csharp
// framework/src/Volo.Abp.Http.Client/Volo/Abp/Http/Client/AbpHttpClientOptions.cs
public class AbpHttpClientOptions
{
    public Dictionary<Type, HttpClientProxyConfig> HttpClientProxies { get; set; }
    public Dictionary<string, List<Action<HttpClientProxyConfig, ClientProxyRequestContext, HttpClient>>>
        ProxyHttpClientPreSendActions { get; }

    public AbpHttpClientOptions AddPreSendAction(string remoteServiceName,
        Action<HttpClientProxyConfig, ClientProxyRequestContext, HttpClient> action)
    {
        ProxyHttpClientPreSendActions
            .GetOrAdd(remoteServiceName, () => new List<…>())
            .Add(action);
        return this;
    }
}
```

The single knob exposed by the Identity domain module: the external-login provider dictionary used by the account flow.

```csharp

public class AbpIdentityOptions
{
    public ExternalLoginProviderDictionary ExternalLoginProviders { get; }
```

```csharp
    public HashSet<string> DeletedPermissionGroups { get; }
```

## AbpSettingOptions

The mirror of `AbpPermissionOptions` for the settings system. Notable extra knob: `ReturnOriginalValueIfDecryptFailed`.

```csharp
// framework/src/Volo.Abp.Settings/Volo/Abp/Settings/AbpSettingOptions.cs
public class AbpSettingOptions
{
    public ITypeList<ISettingDefinitionProvider> DefinitionProviders { get; }
    public ITypeList<ISettingValueProvider> ValueProviders { get; }
```


## Patterns that cut across the catalogue

A few patterns appear over and over in the table above.

<AccordionGroup>
  <Accordion title="Lists of typed providers (ITypeList<T>)">
    `AbpPermissionOptions`, `AbpSettingOptions`, `AbpFeatureOptions`, `AbpDistributedEventBusOptions`, and `AbpLocalizationOptions` all carry one or more `ITypeList<T>` collections. Modules `Add<TProvider>()` to these lists during `ConfigureServices`; ABP discovery picks them up at runtime.

    ```csharp
    Configure<AbpSettingOptions>(options =>
    {
        options.DefinitionProviders.Add<MyProjectNameSettingDefinitionProvider>();
    });
    ```
  </Accordion>
  <Accordion title="Deleted-name hash sets">
    `AbpPermissionOptions.DeletedPermissions`, `AbpSettingOptions.DeletedSettings`, and `AbpFeatureOptions.DeletedFeatures` let you *retire* an identifier — useful when you remove a permission/setting from your definition provider but want the framework to clean up historical grants/values during DB seeding.
  </Accordion>
  <Accordion title="Dictionaries with named entries (Containers / RemoteServices / Inboxes)">
    `AbpBlobStoringOptions.Containers`, `AbpRemoteServiceOptions.RemoteServices`, `AbpDistributedEventBusOptions.Outboxes` and `.Inboxes`, and `AbpDistributedEntityEventOptions.EtoMappings` are all dictionaries keyed by a logical name. They map cleanly onto `appsettings.json` objects when you bind them via `configuration.GetSection(...)`.
  </Accordion>
  <Accordion title="Boolean master-switches">
    `AbpMultiTenancyOptions.IsEnabled`, `AbpAuditingOptions.IsEnabled`, `AbpAntiForgeryOptions.AutoValidate`. Use these to turn a subsystem off (almost) entirely for a single deployment without removing the module.
  </Accordion>
  <Accordion title="HideErrors flags">
    `AbpAuditingOptions.HideErrors` (defaults `true`) and `AbpDistributedCacheOptions.HideErrors` (defaults `true`) tell the subsystem to *log and swallow* internal errors instead of bubbling them. Flip to `false` in tests so failures aren't silent.
  </Accordion>
</AccordionGroup>
