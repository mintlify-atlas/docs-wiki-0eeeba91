The CMS Kit and Blogging modules ship two layers of client assets: the **admin** side (Markdown editing, slug generation, JSTree navigation, file uploading) and the **public** side (syntax-highlighted code blocks, star-rating widgets, image carousels). On npm this is split across four meta packs and seven leaf packs, all under the `@abp/*` scope.

This page documents every pack involved:

- `@abp/cms-kit` вҖ” meta-pack (admin + public)
- `@abp/cms-kit.admin` вҖ” admin meta-pack (CodeMirror, JSTree, Markdown-it, Slugify, TUI Editor, Uppy)
- `@abp/cms-kit.public` вҖ” public meta-pack (Highlight.js, Star-Rating-SVG)
- `@abp/blogging` вҖ” Blogging module meta-pack (TUI Editor, PrismJS, Owl Carousel)
- Leaf packs cited above: `@abp/markdown-it`, `@abp/slugify`, `@abp/tui-editor`, `@abp/star-rating-svg`, `@abp/uppy`, `@abp/jstree`, `@abp/owl.carousel`, `@abp/prismjs`

The C# modules these pair with live in `modules/cms-kit/`, `modules/cms-kit-admin/`, `modules/cms-kit-public/`, and `modules/blogging/`.

## Meta packs

### `@abp/cms-kit`

```json
{
  "version": "10.0.1",
  "name": "@abp/cms-kit",
  "publishConfig": { "access": "public" },
  "dependencies": {
    "@abp/cms-kit.admin":  "~10.0.1",
    "@abp/cms-kit.public": "~10.0.1"
  },
  "license": "LGPL-3.0",
  "homepage": "https://abp.io"
}
```

A pure meta-pack: pulls both halves of the CMS Kit client stack in one go. Used by projects that host both the admin and the public CMS pages in the same web app.

### `@abp/cms-kit.admin`

```json
{
  "version": "10.0.1",
  "name": "@abp/cms-kit.admin",
  "publishConfig": { "access": "public" },
  "dependencies": {
    "@abp/codemirror":  "~10.0.1",
    "@abp/jstree":      "~10.0.1",
    "@abp/markdown-it": "~10.0.1",
    "@abp/slugify":     "~10.0.1",
    "@abp/tui-editor":  "~10.0.1",
    "@abp/uppy":        "~10.0.1"
  }
}
```

What it stages (via the listed leaf packs):

| Feature on admin UI | Leaf pack | Files written to `wwwroot/libs/` |
| --- | --- | --- |
| Markdown source editor | `@abp/codemirror` | `codemirror/{lib,mode/**,theme/**,addon/**}/*` |
| Page / tag tree navigation | `@abp/jstree` | `jstree/**/*.*` |
| Markdown вҶ’ HTML preview rendering | `@abp/markdown-it` | `markdown-it/markdown-it.min.js` |
| URL slug generation for content | `@abp/slugify` | `slugify/slugify.js` |
| Rich Markdown editor (Toast UI) | `@abp/tui-editor` | `tui-editor/*.*` (ABP-authored shim) |
| File / image uploader | `@abp/uppy` | `uppy/*.*` |

### `@abp/cms-kit.public`

```json
{
  "version": "10.0.1",

  "publishConfig": { "access": "public" },

```

| Feature on public site | Leaf pack | Files written to `wwwroot/libs/` |
| --- | --- | --- |
| Syntax-highlighted code blocks (in user comments, blog posts) | `@abp/highlight.js` | `highlight.js/highlight.min.js`, `highlight.js/styles/**` |
| Star-rating widget for blog post / page rating | `@abp/star-rating-svg` | `star-rating-svg/js/*.min.js`, `star-rating-svg/css/*.css` |

### `@abp/blogging`

```json
{
  "version": "10.0.1",
  "name": "@abp/blogging",
  "publishConfig": { "access": "public" },
  "dependencies": {
    "@abp/aspnetcore.mvc.ui.theme.shared": "~10.0.1",
    "@abp/owl.carousel": "~10.0.1",
    "@abp/prismjs":      "~10.0.1",
    "@abp/tui-editor":   "~10.0.1"
  }
}
```

The Blogging module wants everything from the theme.shared umbrella *plus* three blog-specific libraries:

| Blogging feature | Leaf pack | Files written to `wwwroot/libs/` |
| --- | --- | --- |
| Image carousel on the blog landing page | `@abp/owl.carousel` | `owl.carousel/**/*.*` |
| Syntax-highlighted code in posts | `@abp/prismjs` | `prismjs/**/*.*` |
| Post editor on the admin side | `@abp/tui-editor` | `tui-editor/*.*` |

Plus everything `theme.shared` already provides ([packs/theme-shared](/packs/theme-shared)).

## Leaf packs вҖ” package.json + mapping

### `@abp/markdown-it`

```json
{
  "name": "@abp/markdown-it",
  "version": "10.0.1",
  "dependencies": { "@abp/core": "~10.0.1", "markdown-it": "^14.1.0" }
}
```

```js
// abp.resourcemapping.js
module.exports = {
    mappings: {
        "@node_modules/markdown-it/dist/markdown-it.min.js": "@libs/markdown-it/"
    }
}
```

Used by CMS Kit's Markdown-rendered widgets (comments, posts) for the JS-side preview.

### `@abp/slugify`

```json
{
  "name": "@abp/slugify",
  "version": "10.0.1",
  "dependencies": { "slugify": "^1.6.6" }
}
```

```js
// abp.resourcemapping.js
module.exports = {
    mappings: {
        "@node_modules/slugify/slugify.js": "@libs/slugify/"
    }
}
```

Drives the auto-slug field in CMS Kit's create-page form: as the title is typed, `slugify` produces the URL-safe identifier. Note that `@abp/slugify` does **not** depend on `@abp/core` вҖ” the upstream package is a tiny standalone ES module.

### `@abp/tui-editor`

```json
{
  "name": "@abp/tui-editor",
  "version": "10.0.1",
  "dependencies": {
    "@abp/jquery":  "~10.0.1",
    "@abp/prismjs": "~10.0.1"
  }
}
```

```js
// abp.resourcemapping.js
module.exports = {
    mappings: {
        "@node_modules/@abp/tui-editor/src/*.*": "@libs/tui-editor/"
    }
}
```

Unusual shape: the mapping reads from `@node_modules/@abp/tui-editor/src/*.*` вҖ” i.e. ABP ships **its own bundled** TUI editor build inside `src/` rather than re-exposing the upstream `@toast-ui/editor` files directly. This is so ABP can pin the Markdown editor build with its own plugins (PrismJS syntax highlighting, `@abp/jquery` integration) without a `webpack` step in the consumer project.

The deps reveal the wiring: `@abp/jquery` (for jQuery-bridge events into the editor) + `@abp/prismjs` (for the code-block highlighter).

## Why `tui-editor` ships its own `src/`

The upstream `@toast-ui/editor` package needs a build step to produce browser-ready ES modules with the configuration ABP wants (specific plugins, the PrismJS highlight bridge). To keep `install-libs` a pure file-copy step (no `webpack`, no `tsc`), ABP performs that build *once* before publishing `@abp/tui-editor` and ships the result inside `npm/packs/tui-editor/src/`. The mapping then reads from `@node_modules/@abp/tui-editor/src/*.*` вҖ” i.e. ABP's own copy, not the upstream's.

`@abp/bootstrap-daterangepicker` follows the same pattern for its Luxon-friendly fork. See [packs/bootstrap-and-datatables](/packs/bootstrap-and-datatables).
