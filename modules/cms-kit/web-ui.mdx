CMS Kit ships ready-to-use Razor Pages and view components for both the admin back-office and the public-facing site. There are four web projects, each with a specific role:

| Project | Role | Folder root |
| --- | --- | --- |
| `Volo.CmsKit.Admin.Web` | Admin CRUD Razor Pages | [`Pages/CmsKit/*`](https://github.com/abpframework/abp/tree/dev/modules/cms-kit/src/Volo.CmsKit.Admin.Web/Pages/CmsKit) |
| `Volo.CmsKit.Public.Web` | Public reading pages + shared view components | [`Pages/Public/CmsKit/*`](https://github.com/abpframework/abp/tree/dev/modules/cms-kit/src/Volo.CmsKit.Public.Web/Pages/Public/CmsKit) |
| `Volo.CmsKit.Common.Web` | Reaction icons, markdown renderer, shared infrastructure | [`Volo.CmsKit.Common.Web`](https://github.com/abpframework/abp/tree/dev/modules/cms-kit/src/Volo.CmsKit.Common.Web) |
| `Volo.CmsKit.Web` | Meta-bundle depending on all three | [`Volo.CmsKit.Web`](https://github.com/abpframework/abp/tree/dev/modules/cms-kit/src/Volo.CmsKit.Web) |

Each `.cshtml` has a sibling `.cshtml.cs` `PageModel`. The pattern is:

```csharp
public class CreateModel : CmsKitAdminPageModel
{
    public CreatePageInputDto ViewModel { get; set; }
    private readonly IPageAdminAppService _pageAdminAppService;

    public CreateModel(IPageAdminAppService pageAdminAppService)
    {
        _pageAdminAppService = pageAdminAppService;
    }

    public void OnGet() => ViewModel = new CreatePageInputDto();

    public async Task<IActionResult> OnPostAsync()
    {
        await _pageAdminAppService.CreateAsync(ViewModel);
        return RedirectToPage("./Index");
    }
}
```

The page model holds the app service and delegates. Forms post the DTO; `RedirectToPage` issues the PRG (Post-Redirect-Get).

### Bundled assets

`CmsKitAdminWebModule` registers JS/CSS bundles:

```csharp
Configure<AbpBundlingOptions>(options =>
{
    options.StyleBundles.Configure(StandardBundles.Styles.Global,
        bundle => bundle.AddContributors(typeof(CmsKitAdminStyleContributor)));

    options.ScriptBundles.Configure(StandardBundles.Scripts.Global,
        bundle => bundle.AddContributors(typeof(CmsKitAdminScriptContributor)));
```

The contributors pull in `markdown-it` and `Prism.js` for the rich content editor on blog-post and page screens.

### Menu

`CmsKitAdminMenuContributor` adds a top-level "CMS" menu group to the admin navbar with sub-items: Pages, Blogs, Blog Posts, Comments, Tags, Menus, Global Resources. Each menu item is permission-gated by `CmsKitAdminPermissions.*` so users see only what they can manage.

### Dynamic routing for pages

Because pages have user-defined slugs (`/about`, `/docs/getting-started`, etc.) — and the same URL might or might not be a CMS page — the public web project uses an ASP.NET Core **dynamic route value transformer** to decide at runtime whether a given URL maps to a page:

```csharp
// modules/cms-kit/src/Volo.CmsKit.Public.Web/Pages/CmsKitPageRouteValueTransformer.cs
public class CmsKitPageRouteValueTransformer : CmsKitDynamicRouteValueTransformerBase
{
    protected IFeatureChecker FeatureChecker { get; }
    protected IPagePublicAppService PagePublicAppService { get; }
    protected IDistributedCache<PageCacheItem> PageCache { get; }

    public CmsKitPageRouteValueTransformer(
        ICurrentTenant currentTenant,
        ITenantConfigurationProvider tenantConfigurationProvider,
        IFeatureChecker featureChecker,
        IPagePublicAppService pagePublicAppService,
        IDistributedCache<PageCacheItem> pageCache)
        : base(currentTenant, tenantConfigurationProvider)
    {
        FeatureChecker = featureChecker;
        PagePublicAppService = pagePublicAppService;
        PageCache = pageCache;
    }

    protected async override ValueTask<RouteValueDictionary> DoTransformAsync(
        HttpContext httpContext, RouteValueDictionary values)
    {
        if (values.TryGetValue("slug", out var slugParameter) && slugParameter is not null)
        {
            // ... lookup page, set values["page"] = "/Public/CmsKit/Pages/Index", values["slug"] = slug
        }
        return values;
    }
}
```

The host wires this up in `Startup`:

```csharp
endpoints.MapDynamicPageRoute<CmsKitPageRouteValueTransformer>("/{slug:minlength(1)}");
endpoints.MapDynamicPageRoute<CmsKitHomePageRouteValueTransformer>("/");
```

For the root URL `/`, `CmsKitHomePageRouteValueTransformer` looks up the default home page through `IPagePublicAppService.FindDefaultHomePageAsync()` and rewrites the route to `/Public/CmsKit/Pages/Index?slug={homePage.Slug}`.

## Public view components — `Volo.CmsKit.Public.Web/Pages/CmsKit/Shared/Components/`

This is the reusable widget library. Each component is a folder with `Default.cshtml` plus a `*ViewComponent.cs` C# class.

```
Pages/CmsKit/Shared/Components/
├── Blogs/BlogPostComment/Default.cshtml         # comment thread on a blog post
├── Commenting/Default.cshtml                    # generic comment thread (any entity)
├── GlobalResources/
│   ├── Script/Default.cshtml                    # injects site-wide <script>
│   └── Style/Default.cshtml                     # injects site-wide <style>
├── MarkedItemToggle/Default.cshtml              # heart/bookmark toggle button
├── PopularTags/Default.cshtml                   # tag cloud
├── Rating/Default.cshtml                        # 1–5 star widget
├── ReactionSelection/Default.cshtml             # 👍/❤/😂 picker
└── Tags/Default.cshtml                          # tag chips on an entity
```

To embed a widget into your own Razor Page:

```cshtml
@await Component.InvokeAsync(typeof(CommentingViewComponent), new
{
    entityType = "BlogPost",
    entityId = Model.BlogPost.Id.ToString()
})

@await Component.InvokeAsync(typeof(RatingViewComponent), new
{
    entityType = "BlogPost",
    entityId = Model.BlogPost.Id.ToString()
})

@await Component.InvokeAsync(typeof(ReactionSelectionViewComponent), new
{
    entityType = "BlogPost",
    entityId = Model.BlogPost.Id.ToString()
})
```

The components fetch their own data via the public app services on render. Each component has client-side JS in `wwwroot/` that POSTs back to the matching public HTTP endpoint when a user interacts.

### Modal: login prompt

### Embedded virtual files

```csharp
Configure<AbpVirtualFileSystemOptions>(options =>
{
    options.FileSets.AddEmbedded<CmsKitCommonWebModule>();
});
```

All `.cshtml` files, JS, CSS, and images in the Common project are embedded into the assembly and served through ABP's virtual file system — so a host doesn't need to copy them into its `wwwroot`.

## Route map summary

```mermaid
flowchart LR
    URL["/about"]
    Root["/"]
    Admin["/CmsKit/Pages"]

    URL -->|matches /{slug}| PageTransform[CmsKitPageRouteValueTransformer]
    Root -->|matches /| HomeTransform[CmsKitHomePageRouteValueTransformer]
    PageTransform --> PublicPage["/Public/CmsKit/Pages/Index?slug=about"]
    HomeTransform --> PublicPage
    Admin --> AdminPage["/CmsKit/Pages/Index<br/>(IPageAdminAppService)"]

    PublicPage -->|server-side fetch| PublicApi["IPagePublicAppService"]
    AdminPage -->|server-side fetch| AdminApi["IPageAdminAppService"]
```

Admin pages use server-side `PageModel.OnGetAsync` to fetch data via the admin app services. Public pages either delegate to the dynamic route transformer (for arbitrary slugs) or use a static path (for blog routes like `/blogs/{blogSlug}`).

## Customizing the UI

Two common customizations:

**Override a view component** — register your own implementation against the same component name:

```csharp
[ViewComponent(Name = "Rating")]
public class MyRatingViewComponent : AbpViewComponent
{
    public IViewComponentResult Invoke(string entityType, string entityId)
        => View("~/Views/Shared/Components/MyRating.cshtml");
}
```

**Override a Razor Page** — drop a file with the same path into your host project; ABP's virtual file system gives the host's filesystem priority over the embedded copy. `Pages/CmsKit/Blogs/Index.cshtml` in your host replaces the embedded one.
