This page is the map of the `abpframework/abp` mono‑repo. ABP is split across roughly nine top‑level directories — framework packages live in `framework/src`, pre‑built modules in `modules/`, project templates in `templates/`, Angular and MVC client packages in `npm/`, build automation in `build/` and `deploy/`, and reference docs in `docs/`. The inventories below come directly from `ls /home/daytona/repos/abpframework/abp/` and `ls /home/daytona/repos/abpframework/abp/framework/src/`. Naming conventions (`Volo.Abp.<Area>.<Layer>`) are explained in [Solution Conventions](/overview/solution-conventions), and the layered responsibilities are in [Layered Architecture](/overview/layered-architecture).

## Top‑level directories

| Path                  | Purpose                                                                 |
| --------------------- | ----------------------------------------------------------------------- |
| `abp_io/`             | Source of the public abp.io site assets (localization etc.).            |
| `apiSpec/`            | Versioned OpenAPI snapshots for ABP modules.                            |
| `build/`              | PowerShell build orchestration — `build-all.ps1`, `build-all-release.ps1`, `test-all.ps1`, shared `common.ps1`. See [Build System](/overview/build-system). |
| `codecov.yml`         | Codecov ignore / threshold settings.                                    |
| `common.DotSettings`  | ReSharper team settings shared by every solution.                       |
| `common.props`        | Shared MSBuild props imported by every framework `.csproj`. Holds `Version`, `LeptonXVersion`, NuGet metadata. |
| `common.test.props`   | Props imported by `*.Tests` / `*.TestBase` projects.                    |
| `configureawait.props`| Injects `ConfigureAwait.Fody` in Release configuration only.            |
| `delete-bin-obj.ps1`  | Recursive `bin`/`obj` cleaner.                                          |
| `deploy/`             | Release pipeline: `1-fetch-and-build.ps1` through `8-download-release-zip.ps1`. |
| `docs/`               | Long‑form documentation source (Markdown). Includes `docs/en/framework/architecture/domain-driven-design/`. |
| `Directory.Build.props` | Repo‑root MSBuild props (detects test projects, injects `coverlet.collector`). |
| `Directory.Packages.props` | Central package versions (`ManagePackageVersionsCentrally=true`). |
| `framework/`          | The framework itself. `framework/src/` holds every `Volo.Abp.*` package; `framework/test/` holds their tests. |
| `global.json`         | Pins SDK `10.0.100` with `rollForward: latestFeature`.                  |
| `latest-versions.json`| Public version history consumed by the CLI / abp.io.                    |
| `LICENSE.md`          | LGPL‑3.0 license text (`PackageLicenseExpression` in `common.props`).   |
| `modules/`            | First‑party application modules (identity, account, audit‑logging, …).  |
| `npm/`                | Two trees: `npm/ng-packs/packages/` (Angular client), `npm/packs/` (MVC / shared static assets). |
| `NuGet.Config`        | Restricts restore to `nuget.org` only.                                  |
| `NuGet.md`            | README packed into every NuGet artifact (see `common.props`).           |
| `nupkg/`              | Local pack output directory used by `deploy/2-nuget-pack.ps1`.          |
| `source-code/`        | Source‑code distribution scaffolding (for the paid source‑code license drop). |
| `studio/`             | Helper assets for ABP Studio.                                           |
| `templates/`          | New‑solution templates rendered by the CLI: `app/`, `app-nolayers/`, `console/`, `maui/`, `module/`, `wpf/`. |
| `test/`               | Cross‑repo integration tests (not per‑package tests).                   |
| `tools/`              | Misc developer tooling.                                                 |

| `*.HttpApi.HostWithIds`                  | API host that *also* hosts the auth server (consolidated mode).               |
| `*.Web.Host`                             | MVC UI host for tiered deployments.                                           |
| `*.Blazor.WebApp` / `.Server` / `.Tiered`| The `9.0+` "Blazor United" host modes including their `.Client` WASM peers.   |

The root `templates/app/aspnet-core/common.props` sets `<AbpProjectType>app</AbpProjectType>` so the CLI can recognize generated projects.
